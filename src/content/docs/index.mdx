Welcome to the Notificator Project documentation.

This docs site helps both non-technical users and developers complete setup and send notifications successfully.

:::tip[Pick the right track]
If you do not write code, follow **User Guide (No Code)** in the sidebar.
If you build integrations, start in **Developer Guide**.
:::

## What You Can Do Here

- Set up local WordPress dashboard notifications without an account or API key
- Optionally connect mobile push and MQTT delivery
- Use the Notificator mobile app on iOS and Android
- Connect supported open-source Notificator devices
- Configure app behavior and notification routing
- Implement API calls with tested request examples

## Choose Your Path

### I am a non-technical user

1. Read [Complete Workflow](/guides/workflow-overview/)
2. Follow [WordPress Plugin Setup](/guides/wordpress-plugin-setup/)
3. Optional: [Create an account](/guides/account-creation/) in the mobile app
4. Optional: [Create an API key](/guides/mobile-api-key-creation/) for mobile push or MQTT

:::note[No-code outcome]
At the end of this path, WordPress events can appear in the dashboard without writing code. Connecting the app is optional and enables mobile push and MQTT.
:::

### I am a developer

1. Start with [Quick Start](/guides/quick-start/)
2. Use [Code Samples](/guides/code-samples/)
3. For WordPress integrations, read [WordPress Custom Events](/guides/wordpress-custom-events/)
4. Review [Public Notify API](/reference/public-notify/)

:::tip[Developer shortcut]
If you already have a `public_client` key, jump directly to [Quick Start](/guides/quick-start/).
:::

## Public API Endpoint

`POST https://api.notificator-project.com`

This endpoint is for custom integrations using a `public_client` key. The WordPress plugin uses the separate Notificator WordPress service automatically; administrators do not need to call that service directly.

## Need Full Technical Overview?

Continue with [Quick Start](/guides/quick-start/), [Code Samples](/guides/code-samples/), and [Public Notify API](/reference/public-notify/) for endpoint behavior and implementation details.
